Teorier om spil, AIDA, Interaktionsdesign og brugervenlighed Interaktionsdesign handler om at undersøge, designe og teste, hvordan brugeren vil anvende et system. Interaktionsdesignere undersøger brugergrænseflader og interesserer sig for brugervenlighed, usability og user experience. Feltet kaldes også HCI - Human Computer Interaction design. Der er 2 retninger, UI og UX. UI står for User interface design og UX står for User Experience design. UI er det der sker på skærmen, mens UX er hele oplevelsen. Hvis man skal sammenligner med en restaurant, så er UI 'maden', mens UX er 'hele oplevelsen af restauranten'. Se evt video med reklame for Meebook. Hvad er UI og UX i forhold til Meebook? Spil er interaktive, dvs. brugeren kan gøre noget og spillet reagerer umiddelbart efter dette. Spil består af en verden, regler og interaktionsmuligheder. I spil er der et element af tilfældighed, ellers bliver de for kedelige og udfordringer. Motivation handler om, hvad der er, der for en spiller til at synes et spil er sjovt og blive ved. Der skal være en konflikt, et element af konkurrence eller et problem. Flow er en teori om motivation. Sværhedsgrad skal hele tiden tilpasses brugerens evner og spilleren skal hele tiden få relevant feedback. Opgave-løsning i spil foregår typisk ved trial and error - prøv dig frem og få feedback eller mønstergenkendelse! Interaktionsdesign, brugervenlighed og AIDA Gense IFTEK-siderne om brugervenlighed.
